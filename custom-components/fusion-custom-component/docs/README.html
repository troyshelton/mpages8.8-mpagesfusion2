MPages Fusion Custom Component Setup

Prerequisites

The project works with MPages Gaia. See Overview of MPages Gaia for more information.

Ensure that the following prerequisites are met before you begin configuring this functionality:

  • Obtain a TOKEN_ID for downloading MPages Gaia and its plugins. Log a Service Request to obtain an MPages Gaia token and the associated MPages Gaia URLs. See eService Help Pages for more information on creating a Service Request.
  • Add the TOKEN_ID in the package.json file.
  • Run npm install.

MPages Gaia is preconfigured in the template. See Install MPages Gaia for more information.

Bedrock Configuration

Install the Bedrock Filters

You must install the Bedrock filters that provide 10 custom component filter means that are compatible with the Fusion custom component API. This only needs to be completed once per domain.

Complete the following steps to install the Bedrock filters:

  1. Copy the .CSV files within bedrock/csv/ into your domain. Typically, this is in this location: /cerner/w_standard/<domain>/install.
  2. Confirm the contents are copied.
  3. In a CCL backend command prompt, compile and run the custom component script br_run_wf_fusion_custom_comp.prg within bedrock/script/ using the following command: 
execute br_run_wf_fusion_custom_comp go

Configure the Bedrock Filters

After importing the Bedrock filters, complete the following steps to configure the MPages View.

  1. In MPages: View Builder (opened from Bedrock.exe), create a new Workflow view and add the Fusion Workflow Custom Components to the view, or add the Fusion Workflow Custom Components to an existing view.
  2. In MPages: ViewPoint Setup (opened from Bedrock.exe), add the view to your ViewPoint.
  3. In MPages: MPages Setup (opened from Bedrock.exe), configure the custom components in the view.

See Configure MPages Views for more information.

The following filter means are used for the custom components in Bedrock and will be used later during the configuration process: 

    FUSION_CUSTOM_COMP_1
    FUSION_CUSTOM_COMP_2

Using Gaia for Custom Components

Extract the compressed file within template/ folder into your project directory to begin working on your component. This template acts as the base for your component.

Install and Configure Gaia

Complete the steps in the following sections to set up the development environment:

Install Gaia

Complete the following steps to install Gaia for use with custom components:

  1. Ensure you have completed the software prerequisites for MPages Gaia. See Software Prerequisites for more information.
  2. Replace <TOKEN_ID> with your MPages Gaia TOKEN_ID in the package.json for mpage-gaia and gaia-preset-public in devDependencies
  3. Run npm install
  4. Run npm run gaia server
  5. Navigate to http://localhost:9001/app

Configure Gaia

See the associated Help link in each Gaia tab for more information. Complete the following steps to configure Gaia for use with custom components:

  1. In the Live Dev tab, add your static content location.
  2. Add the appropriate CCL program. In most cases, this is mp_unified_driver.
  3. Click Param Parser to generate the program parameters.
  4. Select Enable Webpack.
  5. Select Enable Auto-Refresh.
  6. Ensure that the Webpack Target is set to dev.
  7. Click Save Configuration and click Launch.

Replace the Source Code

Before developing your custom component, you must replace all occurrences of MPagesFusionCustomComponent with your component namespace in the following areas:

  • File names
  • Class names
  • Constants
  • Identifier in package.json
  • Entry points in gaia.json

Note: If you need to include multiple components within custom-components.js include an index.js provided with this template and use that file as the entry point in your gaia.json instead.

Using Options property from Bedrock

Because you determine the component’s Bedrock settings, you do not need to depend on the Namespace property. However, the FREE_TEXT option is still available for providing values into the code from Bedrock. Use the Options property by defining the constant in your MPagesFusionCustomComponentConstants.js file and referencing it in your code according to the following example: 

// Bedrock Options property has "optionsObjectPrimary"
// In your MPagesFusionCustomComponentConstants.js you will have:
const CUSTOM_COMPONENT_OPTIONS = {
    optionsObjectPrimary: {
      // ... Omitted for brevity
    },
    optionsObjectSecondary: {
      // ... Omitted for brevity
    }
};

// In your source code MPagesFusionCustomComponent.js you will have:

// Get Bedrock values
loadFilterMappings() {
    this.addFilterMappingObject("FUSION_CUSTOM_COMP_OBJ_1", {
        setFunction: this.setFilterOptions,
        type: "STRING",
        field: "FREETEXT_DESC"
    });
}

// Set the Bedrock values
setFilterOptions(options) {
    this._state.options = options;
    return this;
}

// Get the values from different "Options"

const referencedData = CUSTOM_COMPONENT_OPTIONS[this._state.options].data;

Deploying the Component to the Domain

Run npm run build in the command line while at the component root to create the final compiled component file. This creates a new dist/ folder in your component with the MPagesFusionCustomComponent.js and MPagesFusionCustomComponent.css files. You can add the source code in these files into your custom-components.js and custom-components.css files respectively.
